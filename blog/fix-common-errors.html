How to Fix Common Claude Code Errors (2026)

By Eric Coste March 12, 2026 11 min read

Quick Diagnostic: Run These First

Before diving into specific errors, run these five checks in order. They resolve about 90% of Claude Code issues:

# 1. Check installation
claude --version

# 2. Run built-in diagnostics
claude doctor

# 3. Check your auth status (inside Claude Code)
/status

# 4. Verify Node.js version
node --version  # Needs 18+

# 5. Test network connectivity
curl -s https://storage.googleapis.com > /dev/null && echo "OK" || echo "BLOCKED"

"This organization has been disabled"

This is the single most common Claude Code error, and it's almost always caused by an old API key overriding your subscription. When ANTHROPIC_API_KEY is set in your environment, Claude Code uses that key instead of your Pro/Max subscription's OAuth credentials.

# Fix: Remove the old API key
unset ANTHROPIC_API_KEY

# Remove from shell profile permanently
# Check these files for 'export ANTHROPIC_API_KEY=' lines:
# ~/.zshrc, ~/.bashrc, ~/.profile, ~/.bash_profile

# Re-authenticate
claude auth login

# Verify
/status  # Should show your subscription

Command Not Found

Your terminal can't locate the Claude Code binary. This means either Node.js isn't installed, or the npm global packages directory isn't in your PATH.

# Check Node.js
node --version  # Need 18+

# If missing, install via nvm (recommended)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 20
nvm use 20

# Reinstall Claude Code
npm install -g @anthropic-ai/claude-code

# Verify PATH includes npm global bin
npm config get prefix  # e.g., /usr/local or ~/.npm-global
# Add {prefix}/bin to PATH in your shell profile

TLS/SSL Connection Errors

Errors like curl: (35) TLS connect error or Could not establish trust relationship indicate the connection to Anthropic's servers is being intercepted or blocked — usually by corporate firewalls, proxies, or outdated CA certificates.

# Test direct connection
curl -v https://storage.googleapis.com 2>&1 | head -20

# If behind a corporate proxy:
export HTTPS_PROXY=http://your-proxy:8080
export HTTP_PROXY=http://your-proxy:8080

# Update CA certificates (Linux)
sudo apt update && sudo apt install ca-certificates

Architecture Mismatch

If you see errors about loading binaries, dyld: cannot load, or Abort trap: 6, the downloaded binary doesn't match your system architecture (ARM vs x86_64) or your OS version.

# Check your architecture
uname -m
# x86_64 = Intel/AMD 64-bit
# aarch64/arm64 = ARM (Apple Silicon, ARM servers)

# On macOS, check version (need 13.0+)
sw_vers -productVersion

# Reinstall with correct architecture
npm uninstall -g @anthropic-ai/claude-code
npm install -g @anthropic-ai/claude-code

Context Window Overflow

When your conversation fills the context window, Claude's responses degrade — it forgets earlier instructions, repeats itself, or produces lower-quality code. Claude auto-compacts at 95% capacity, but proactive management is better.

# Check current usage (inside Claude Code)
/cost

# Compress while keeping important context
/compact Keep the implementation approach and API signatures

# Start completely fresh
/clear

# For persistent instructions, use CLAUDE.md instead of
# repeating them in conversation

MCP Server Connection Failures

MCP servers extend Claude Code with external tools, but connection issues are common. The most frequent cause is misconfigured server paths or authentication.

# Debug MCP connections
claude --mcp-debug

# Check your MCP config
cat .claude/settings.json | jq '.mcpServers'

# Common fix: restart both MCP server and Claude Code
# Verify the server process is running
ps aux | grep mcp

Sandbox Errors

If sandboxing fails with namespace creation errors, you're likely in an environment that doesn't support user namespaces (Docker, WSL1, or an older kernel).

# Check sandbox status (inside Claude Code)
/sandbox

# On Linux, install requirements
sudo apt install bubblewrap socat

# In Docker, enable weaker sandbox mode
# In your settings.json:
# "sandbox": { "enableWeakerNestedSandbox": true }

# WSL1 is not supported — upgrade to WSL2
wsl --set-version Ubuntu 2

Rate Limit Errors (429)

HTTP 429 means you've hit your usage limits. For Pro subscribers, this is the 5-hour sliding window. For API users, it's the requests-per-minute limit.

# Check your current plan limits
/status

# Quick mitigations:
# 1. Switch to a cheaper model for routine tasks
/model sonnet

# 2. Reduce thinking token budget
export MAX_THINKING_TOKENS=8000

# 3. Wait for the rate limit window to reset
# Pro: ~5 hour window, Max 5x: 5x the window

Frequently Asked Questions

Why does Claude Code say my organization is disabled?

This happens when an old ANTHROPIC_API_KEY environment variable overrides your subscription authentication. Run 'unset ANTHROPIC_API_KEY' and remove it from your shell profile (~/.zshrc or ~/.bashrc), then authenticate with 'claude auth login'.

How do I fix Claude Code command not found?

This means Node.js isn't installed or the npm global bin directory isn't in your PATH. Verify with 'node --version' (needs 18+) and check your PATH includes the npm global directory. On macOS/Linux: 'npm config get prefix' then add /bin to your PATH.

Why is Claude Code running slowly?

Long conversations accumulate context, slowing responses. Use '/compact' to summarize, '/clear' when switching tasks, and check your model — Opus is slower than Sonnet. Also check status.claude.com for service issues.

What does the context window error mean?

You've filled Claude's context window (200K or 1M tokens depending on model). Claude auto-compacts at 95% capacity, but you can manually run '/compact' with instructions about what to preserve, or '/clear' to start fresh.
