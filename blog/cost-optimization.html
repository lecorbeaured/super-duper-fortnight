Claude Code Cost Optimization: Reduce API Spend by 50-75% (2026)

By Eric Coste March 1, 2026 10 min read

Where Your Tokens Go

Understanding token consumption is the first step to reducing it. Claude Code costs come from four sources: input tokens (your messages + file reads + context), output tokens (Claude's responses + code + thinking), cache writes (first-time context storage), and cache reads (subsequent context retrieval).

The surprising reality: over 90% of tokens for most developers are cache reads. Input and output tokens — the ones you'd normally think about — are a tiny fraction of the total. But output tokens cost 5x more per token than input, so optimizing what Claude generates has outsized impact.

The Four Highest-Impact Strategies

These four changes account for the majority of achievable savings. Implement them in order of impact.

1. Cap thinking tokens (biggest lever)

# Set in your shell profile
export MAX_THINKING_TOKENS=10000

# Or per-session: reduce effort level
/effort low    # For simple tasks
/effort medium # Default
/effort high   # For complex reasoning

Extended thinking is enabled by default and can consume tens of thousands of tokens per response. For routine coding tasks, 10,000 tokens provides sufficient reasoning depth. This single change can reduce costs by 30-40%.

2. Model selection (use Sonnet by default)

# Default to Sonnet for implementation
/model sonnet

# Switch to Opus only for architecture and complex debugging
/model opus

# Or use opusplan: Opus for planning, Sonnet for code
/model opusplan

Sonnet costs $3/$15 per million tokens. Opus costs $15/$75. That's a 5x difference. Sonnet handles 80% of coding tasks just as well. Reserve Opus for complex planning and deep analysis.

3. Context management

# Clear between unrelated tasks — stale context wastes tokens
/clear

# Compact with specific instructions
/compact Keep only the API contract and current implementation

# Use CLAUDE.md for persistent context instead of
# repeating it in every conversation

Every message you send includes the entire conversation history. A 50-message session costs significantly more per message than starting fresh. Aggressive clearing and compaction keeps per-message costs low.

4. Specific prompting

Vague prompts force Claude to explore, reading many files to understand what you want. Specific prompts with file names, expected outcomes, and constraints let Claude act directly. The tokens saved on exploration compound across hundreds of interactions per month.

Subscription vs API: The Math

If your monthly API equivalent exceeds $100, Max 5x ($100/month) saves money. If it exceeds $200, Max 20x ($200/month) makes sense. Both include Opus access and priority during peak demand.

Track your API spend for one week, multiply by 4, and compare against subscription tiers. Most daily Claude Code users save 50-90% by switching from API to Max.

Advanced Cost Reduction

Beyond the big four, these techniques provide incremental savings: use subagents for verbose operations (tests, docs, log analysis) so output stays in the subagent's context, disable non-essential traffic with CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1, and keep CLAUDE.md under 50 lines — every instruction consumes tokens on every message.

Monitoring Your Spend

# Session cost (inside Claude Code)
/cost

# Historical analysis
npx ccusage

# Enable cost in your status bar
# Settings > Status line > Show cost

You can't optimize what you don't measure. Track your weekly spend, identify which sessions cost the most, and adjust your model selection and session management accordingly. Most developers reduce costs 40-70% within the first month of active monitoring.

Frequently Asked Questions

How much does Claude Code cost per month?

Average is $6/day or $100-200/month on API with Sonnet. Pro subscription is $20/month with usage limits. Max 5x ($100/month) and Max 20x ($200/month) offer higher limits. Most developers save money on Max vs API if they use Claude Code daily.

What's the single biggest cost saving tip?

Set MAX_THINKING_TOKENS to 10,000. Extended thinking can consume tens of thousands of tokens per response, and for most routine development tasks, 10K tokens provides sufficient reasoning depth. This alone can reduce costs 30-40%.

Should I use Sonnet or Opus?

Default to Sonnet for 80% of tasks — it handles most coding work well at 1/5 the cost of Opus. Switch to Opus only for complex architecture decisions, deep debugging, or multi-system refactoring where reasoning quality matters.

How do I track my Claude Code spending?

Use /cost inside Claude Code for session stats. Install ccusage (npm package) for historical analysis. On API, use Anthropic's Usage & Cost Admin API. Max subscribers can check /stats for usage patterns.
